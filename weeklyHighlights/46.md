## 1.1 新Fun-ASR-Realtime – 阿里千问推出的流式实时语音识别模型

Fun-ASR-Realtime是阿里千问推出的流式实时语音识别大模型，基于WebSocket协议实现边说边出字，首字延迟低至百毫秒。它支持普通话、16种方言及30种语言，具备上下文纠错、情感识别等能力，能满足直播、会议等多场景的低延迟语音转写需求。

#### 1.1.1 核心功能
1. 实时流式识别：通过WebSocket双工通信实现音频到文本的低延迟转换，支持边说边出字。
2. 多方言多语种支持：覆盖16种方言和30种语言，适配出海客服、国际会议等跨语言场景。
3. 上下文纠错：基于历史对话和热词自动修正识别结果，提升专业词汇、特定名称的识别准确率。
4. 情感识别：支持细粒度或极性情感分类，可同步检测说话人的情绪状态。
5. 时间戳输出：默认输出句级与字级时间戳，便于字幕对齐和关键词定位。

#### 1.1.2 技术原理
采用Transformer/Conformer骨干网络构建流式大模型架构，通过流式推理机制实现音频边输入边处理。运用因果注意力掩码和分块输入技术，平衡延迟与识别准确率；基于泛Context强化训练和上下文感知重打分机制，实现语境化纠错。采用多任务联合建模，共享编码器同时完成语音识别、情感识别和VAD断句任务；通过CTC或注意力对齐路径实现字级/句级时间戳同步输出。

#### 1.1.3 应用场景
1. 直播实时字幕：为长时直播、多人互动场景提供低延迟字幕输出，支持说话人频繁切换，适配复杂直播环境。
2. 会议实时转写：在远程会议、线下论坛中实现边说边出字，输出带时间戳的结构化文本，便于会后回溯关键内容。
3. 客服通话质检：实时识别客服与用户对话，同步检测情感状态，结合热词定制捕捉关键信息，实现实时风险预警。
4. 语音输入法：为移动端、车载设备提供多方言实时转写能力，提升非普通话用户的输入效率与体验。

## 1.2 新Leanstral 1.5 – Mistral AI 开源的形式化验证大模型

Leanstral 1.5是Mistral AI开源的Apache 2.0许可形式化验证大模型，基于119B参数MoE架构，专为Lean 4自动定理证明优化，支持256k超长上下文与图文输入。它在miniF2F、PutnamBench等数学证明基准达SOTA，能自动完成复杂定理证明与代码正确性验证，成本远低于同类模型，推动AI形式化验证的普及与自动化。

#### 1.2.1 核心功能
1. 自动定理证明：支持从初等数学到IMO级、Putnam竞赛级的定理证明，在miniF2F实现100%饱和解决，PutnamBench解决587/672问题。
2. 代码属性验证：可证明代码的时间复杂度、逻辑正确性等核心属性，如为AVL树实现完成O(log n)时间复杂度的形式化证明。
3. 自动化bug检测：通过生成并验证代码正确性属性，能发现传统测试易遗漏的边缘性bug，曾在57个仓库中找出5个未公开bug。
4. 长上下文推理：支持最长4M tokens的推理流程，可完成跨文件、多轮迭代的长周期证明工程任务。
5. 工具调用协作：可与Lean语言服务器、文件系统等工具交互，模拟开发者工作流完成复杂证明工程。

#### 1.2.2 技术原理
采用119B参数稀疏激活MoE架构，每token仅激活约6B参数，平衡性能与推理成本。训练流程分为三阶段：先通过mid-training在大规模数学与代码语料上预训练，再经监督微调对齐Lean 4证明范式，最后通过CISPO强化学习在多轮交互环境中优化。训练环境包含多轮定理证明环境（接收Lean编译器反馈迭代优化证明）和代码代理环境（模拟开发者操作文件、调用工具完成长周期任务）。推理时支持FLASH_ATTN_MLA注意力后端，配合vLLM实现高效的长上下文推理与工具调用解析。

#### 1.2.3 应用场景
1. 数学研究辅助：面向数学研究者，自动生成定理证明草稿、辅助完成复杂引理证明，加速数学研究进展。
2. 高可靠代码开发：面向军工、金融等领域的高可靠软件开发者，自动证明代码的安全性、正确性属性，提前发现隐藏bug。
3. 计算机科学教学：面向高校计算机专业师生，用于形式化方法课程教学，辅助学生理解定理证明逻辑与代码验证方法。
4. 开源项目质量保障：面向开源项目维护者，批量验证代码正确性，提升项目可靠性，降低人工审核成本。
5. 形式化知识库构建：面向AI研究人员，自动将数学定理、代码规范转化为形式化知识库，为通用AI提供可靠的逻辑推理基础。

## 1.3 新Elements Claw – 阿里达摩院推出的超导材料发现 AI 智能体

Elements Claw是阿里达摩院推出的超导材料发现AI智能体，采用专通融合架构，将10亿参数原子基础模型Elements与大型语言模型结合，实现从文献检索、性质预测到实验方案设计的全流程自动化材料筛选。它仅用28个GPU小时即完成240万晶体结构筛选，预测出6.8万个超导候选材料，还成功实验验证4种全新超导材料，能大幅缩短材料研发周期。

#### 1.3.1 核心功能
- **超导临界温度精准预测**：基于Elements-T模型，可精准预测材料的超导临界温度Tc值，平均误差控制在1K以内，为筛选高潜力超导材料提供关键依据。
- **超导性高效二元分类**：通过Elements-C模型判断材料是否具有超导性，AUC高达0.996，能快速从大量材料中筛选出可能的超导材料。
- **热力学稳定性科学评估**：利用Elements-E模型评估候选材料的合成可行性与结构稳定性，帮助判断材料是否具备实际合成价值。
- **晶体结构从头设计生成**：借助Elements-G模型实现晶体结构的从头设计，支持 motif引导与全新相探索，扩展潜在超导材料的结构范围。
- **文献智能综合提炼**：依托LLM自动检索、阅读并提炼科学文献，挖掘分散的实验知识，为材料研究提供全面的文献支撑。
- **实验方案专业设计**：综合数值计算与语义推理，输出可执行的合成路径与验证策略，指导实验人员开展材料合成与验证工作。

#### 1.3.2 技术原理
- **原子基础模型架构**：Elements模型基于EquiformerV2等变架构，在1.25亿分子和晶体结构上预训练，实现周期晶体与非周期分子的统一表示，通过多任务预训练策略，同时掌握稳定物质的静态结构特征与驱动结构演化的动态力场。
- **专通融合Agent架构**：专有模型层（Elements-T/C/E/G）负责原子尺度的数值计算，通用LLM层负责高层语义推理、文献综合与决策规划，两者通过函数调用协调，由LLM编排任务链，自适应组合内外部工具。
- **自我进化机制**：Agent可从文献中提炼新见解，通过微调Elements自主创建新技能工具（如Elements-C），实现“发现需求—制造工具—验证效果”的闭环能力扩展。
- **生成式晶体结构预测**：采用先验 informed 扩散策略，直接在笛卡尔坐标系中进行扩散和去噪，利用Elements模型作为编码器，提升晶体结构生成的准确性和效率。

#### 1.3.3 应用场景
- **超导材料研发**：科研人员通过输入目标元素或体系（如Hf-Zr-Re），利用该智能体快速筛选出高Tc值的超导候选材料，大幅缩短超导材料的研发周期，加速新型超导材料的发现。
- **材料数据库构建**：依托其大规模晶体结构筛选能力，可构建包含6.8万+潜在超导材料的数据库，为材料科学研究提供丰富的数据资源，推动相关领域的研究进展。
- **实验方案优化**：针对特定候选材料，智能体可生成专业的实验合成路径与验证策略，帮助实验人员优化实验方案，提高实验成功率，降低研发成本。
- **跨体系材料探索**：支持对不同元素组合的体系进行超导材料探索，如Zr-Sc-Re、Zr-V-Re等，助力发现跨体系的新型超导材料，拓宽超导材料的研究范围。

## 1.4 ComAct – 上海AI Lab等推出软件自动化操作的研究范式

ComAct是由上海AI Lab等机构提出的专业软件自动化操作研究范式，以Windows的组件对象模型（COM）为统一可执行抽象，将专业软件交互从脆弱的视觉控制转变为确定性程序合成，解决GUI智能体长任务错误累积和API方法适配性差的问题。配套的ComActor智能体在ComCADBench工业CAD基准测试上超越GPT-5等模型，实现长流程任务的高成功率执行。

#### 1.4.1 核心功能
1. **COM代码生成执行**：将自然语言指令转化为可执行的COM Python脚本，直接调用专业软件底层接口，替代多步GUI操作避免误差累积。
2. **多模态自修正机制**：结合终端执行反馈与界面截图，诊断代码错误并迭代修正，提升任务执行成功率。
3. **跨专业软件适配**：统一操作SolidWorks、AutoCAD等CAD软件及Office、Adobe套件，支持跨应用工作流自动化。
4. **大规模并行训练**：依托ComForge平台的Docker化Windows容器集群，实现智能体的高效并行训练与测试。
5. **几何精度对齐优化**：通过Chamfer距离计算几何相似度，结合GRPO强化学习优化，保障生成结果的工程精度。

#### 1.4.2 技术原理
基于部分可观测马尔可夫决策过程（POMDP）建模专业软件操作，将每个动作定义为COM脚本执行，通过"思考-决策-执行"闭环完成任务。智能体采用三阶段训练框架：第一阶段基于验证的指令-代码对进行文本到代码的监督微调（SFT），掌握COM API的基础调用能力；第二阶段引入多模态交互轨迹，实现错误诊断与代码修正的agentic SFT；第三阶段采用Group Relative Policy Optimization（GRPO）强化学习，以连续几何奖励对齐任务级精度。ComForge平台提供标准化接口，支持代码执行、GUI操作和环境重置，异步调度器实现千级容器并行扩展，对接ms-swift框架完成全流程训练。

#### 1.4.3 应用场景
1. **工业CAD自动化设计**：面向机械工程师，自动完成3D建模、工程图绘制、装配干涉检测等复杂CAD任务，支持SolidWorks、Inventor等主流软件。
2. **跨专业软件工作流**：实现从CAD建模到Office文档生成、Adobe图纸渲染的全流程自动化，适用于产品设计与文档输出一体化场景。
3. **大规模CAD模型生成**：依托并行训练能力，批量生成符合工程规范的3D模型，满足工业仿真、数字孪生等场景的模型需求。
4. **CAD基准测试与算法验证**：基于ComCADBench基准数据集，为CAD生成算法提供标准化测试环境，支持模型性能的定量对比分析。

## 1.5 Vidu S1  – 生数科技推出的实时交互视频基础模型

Vidu S1是生数科技推出的实时交互视频基础模型，标志AI视频进入实时双向互动时代。它基于自回归扩散架构，支持540P分辨率、最高42FPS的实时视频生成，可在消费级GPU运行。用户上传单张图片即可创建数字角色，通过语音指令实时驱动角色全身动作，实现无限时长稳定互动，适用于AI陪伴、虚拟偶像等多场景。

#### 1.5.1 核心功能
1. 语音指令驱动：支持通过自然语音指令实时控制数字人的表情、手势及全身动作，实现高精度行为跟随。
2. 无限时长生成：全球首个支持无限互动时长的生成式视频大模型，可数小时连续生成且保持角色身份稳定。
3. 自定义角色创建：用户上传真人、动漫或萌宠图片即可零训练创建专属数字角色，还支持音色选择与克隆。
4. 实时高清交互：提供540P/25FPS（最高42FPS）的高清实时视频互动，达到视频通话级别的流畅度。
5. 场景感知响应：具备场景理解能力，可感知摄像头画面并实时调整互动内容，适配动态环境。

#### 1.5.2 技术原理
采用自回归扩散架构，训练阶段分为三步：先训练双向教师模型建立高质量生成先验，再通过Teacher Forcing与Diffusion Forcing混合策略将模型适配为因果自回归架构，最后结合Distribution Matching Distillation（DMD）与Phased Consistency Models（PCM）正则化实现三步采样的高效推理。推理阶段采用滑动窗口解码结合TwinCache缓存策略，同时搭配TurboDiffusion与TurboServe加速栈，通过SageAttention、算子融合、CUDA Graph等技术实现消费级GPU上的实时推理。

#### 1.5.3 应用场景
1. 虚拟偶像运营：娱乐公司可快速打造专属虚拟偶像，通过语音指令实时控制其舞台动作、直播互动，降低内容制作成本。
2. 游戏智能NPC：游戏厂商为NPC接入Vidu S1，玩家可通过语音与NPC实时互动，获取任务线索或触发剧情，提升游戏沉浸感。
3. AI陪伴助手：面向C端用户提供个性化AI陪伴，用户可自定义陪伴角色形象，通过语音交流获得情绪支持、知识解答等服务。
4. 互动直播带货：主播可创建数字人分身，通过语音指令控制其讲解产品、展示动作，实现24小时不间断直播带货。

## 2.13 TabFM – 谷歌开源的零样本表格基础模型

TabFM是Google Research推出的零样本表格基础模型，支持混合数值与类别特征的表格数据分类（最多10类）和回归任务。它基于数百亿合成表格数据预训练，无需微调或超参数搜索，通过上下文学习单次前向预测即可完成任务，在TabArena基准51个数据集上表现优于经大量调优的梯度提升树等模型，还提供PyTorch与JAX双后端版本，兼容scikit-learn接口。

1. 零样本分类：无需训练数据调参，直接对最多10个类别的表格数据完成分类预测，降低使用门槛。
2. 零样本回归：支持数值型目标变量的回归任务，无需模型微调，可快速输出连续值预测结果。
3. 混合特征处理：原生支持数值与类别混合特征的表格数据，自动完成编码、缩放等预处理操作。
4. 上下文学习推理：将训练数据作为上下文输入，通过单次前向传播完成预测，无需更新模型权重。
5. 双框架兼容：提供JAX与PyTorch两种权重版本，适配不同技术栈的开发与部署环境。

#### 2.13.2 技术原理
采用混合Transformer架构，包含交替行列注意力模块、行压缩层和ICL Transformer三部分：首先通过3层列注意力（4头、256诱导点）和3层行注意力（8头、8个CLS token）捕捉特征交互与行级模式；接着将每行压缩为稠密向量；最后由24层ICL Transformer（8头）处理压缩后的向量序列实现上下文学习。基于结构因果模型（SCM）生成数百亿合成表格数据预训练，规避真实数据隐私问题，同时通过SVD特征、特征交叉和NNLS融合的Ensemble方案进一步提升性能。

2. 医疗辅助诊断：医护人员基于患者的混合特征数据进行疾病分类，无需接触敏感原始医疗数据。

AniShort 是面向 AI 短剧生产的协作平台，官方页面标题为 “AI Short Drama Collaboration Platform”。它把短剧创作从单人提示词生成扩展到多人协同与流程化生产，更适合需要持续产出短剧项目的团队。与单点视频生成工具相比，AniShort 的价值在于围绕短剧项目管理、内容协作和生产链路组织提供一个统一入口。

#### 2.25.1 核心功能
1. 短剧项目协作：面向剧本、角色、分镜、素材和视频生成流程提供统一协作空间。
2. AI 辅助创作：围绕短剧内容生产提供 AI 生成、修改和迭代能力，减少人工拆解脚本和镜头的成本。
3. 团队流程管理：支持多人围绕同一短剧项目协同推进，适合短剧工作室和内容团队。
4. 跨环节资产沉淀：将角色、场景、脚本和镜头素材集中管理，便于复用和维护一致性。

#### 2.25.2 技术原理
AniShort 采用前端工作台承载短剧项目流，把 AI 生成能力嵌入协作流程中。系统重点不是单次生成视频，而是把短剧生产中的文本创作、角色设定、场景资产、镜头生成和团队协同组织起来，让 AI 生成结果可以在项目维度反复修改、复用和交付。

#### 2.25.3 应用场景
1. 短剧工作室：多人协作完成 AI 短剧从脚本到分镜、成片的生产流程。
2. 内容团队：围绕多个短剧选题并行推进，统一管理角色和素材资产。
3. 创作者社区：用协作平台降低短剧项目组织成本，把个人创意变成可分工执行的项目。

* 官网：https://anishort.ai/

## 2.26 Zopia - 面向营销人与创作者的 AI 视频与图像 Agent

Zopia 是面向营销人与创作者的多模型 AI 视频与图像平台，官方定位为 “AI Video & Image Agent”。它支持从文字、图片或视频生成 UGC 广告、品牌片和社交视频，并内置 Director Agent，将脚本编辑、角色设置、故事板生成和最终剪辑整合进一个自动化工作流。

#### 2.26.1 核心功能
1. Director Agent：输入创意或脚本后，自动处理脚本编辑、角色设置、故事板生成和最终剪辑。
2. 多模型生成平台：统一接入 Seedance 2.0、Kling、Wan、CogVideo、Flux、Minimax、HunyuanVideo 等视频和图像模型。
3. UGC 广告生成：面向营销场景批量生成广告创意，支持快速测试 hook、素材风格和投放方向。
4. 品牌片与社交视频：从文本、图片或视频生成品牌片、短视频和社媒内容。
5. OpenClaw API：通过统一 API 调用 Zopia 的生成流水线，方便开发者把视频 Agent 能力集成进自己的产品。

#### 2.26.2 技术原理
Zopia 将多模型路由、Agent 编排和视频编辑流程结合在一起。Director Agent 负责把用户意图拆解为脚本、角色、分镜和剪辑任务，多模型层负责根据不同任务调用适合的视频或图像模型，工作流层则把生成结果串联为可交付的视频内容。它的重点是让创作者不必在多个模型和工具之间手动切换，而是在一个连续流程中完成从创意到成片的生产。

#### 2.26.3 应用场景
1. 广告投放团队：快速生成多版本 UGC 广告素材，缩短创意测试周期。
2. 品牌内容团队：制作品牌片、产品宣传视频和社交媒体短片。
3. 独立创作者：用一个工作台完成脚本、分镜、生成和剪辑，降低视频制作门槛。
4. 开发者与自动化团队：通过 OpenClaw API 将视频生成和 Agent 流程嵌入内部工具或 SaaS 产品。

* 官网：https://zopia.ai/zh
