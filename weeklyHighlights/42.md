## 1.1 新Kimi Work 媒介投放实战 – Agent集群自动抓取数据与分析

Kimi Work是运行在本地电脑的Agent团队工具，获授权后可调用本地文件、操作电脑，搭配专属浏览器插件能以用户登录态爬取数据。它通过多Agent集群并行协作，可高效完成媒介投放数据复盘等繁琐工作，大幅节省人力时间成本。

#### 1.1.1 核心功能
1. 本地资源调用：获授权后可直接调用、修改本地存储的各类文件，无需手动上传导入。
2. 集群Agent协作：支持多Agent并行工作，最高可调度300个Agent同步处理任务，提升工作效率。
3. 登录态数据爬取：专属浏览器插件可携带用户登录态爬取数据，规避网站爬虫拦截，获取更多操作权限。
4. 多角色任务分工：可拆分数据采集、分析、归因等不同任务，由对应Agent专项处理，输出专业结案数据和PPT。

#### 1.1.2 技术原理
基于Agent集群架构，通过任务调度系统分配并行任务，实现多Agent协同工作；浏览器插件采用用户登录态复用技术，模拟真实用户访问行为，绕过反爬虫机制；结合OCR图像识别技术，可解析用户提供的数据截图提取有效信息；本地资源调用基于系统权限授权机制，实现对本地文件系统的安全访问与操作。

#### 1.1.3 应用场景
1. 营销投放复盘：营销人员提供投放清单，工具自动抓取并分析数据，输出结案数据表和PPT，替代手动统计分析。
2. 多平台数据汇总：适需从多个网站采集数据的场景，如竞品信息收集、行业数据汇总，由Agent批量爬取整理。
3. 复杂文档处理：对本地多格式文档进行批量修改、内容提取、格式转换等操作，提升文档处理效率。

## 1.4 PawBench – 阿里通义推出的通用智能体评测基准

PawBench是通义实验室推出的通用智能体评测基准，将底座大模型与Agent运行框架（Harness）纳入同一评测体系。它构建了150道真实任务的评测集，通过多模型与多框架的交叉评测，能精准定位框架问题，帮助开发者优化Harness，为智能体选型提供数据支持。

#### 1.4.1 核心功能
1. 联合评测：将底座模型与运行框架纳入同一体系，评估LLM×Harness的联合运行效果，明确两者的性能贡献占比。
2. 五维标签体系：对任务按应用场景、原子能力、复杂度、输入模态、运行环境进行结构化标注，支持多维度切片分析。
3. 混合评分机制：结合自动规则断言与LLM-as-judge语义评估，在保证客观性的同时覆盖复杂语义场景，分数范围0-1。
4. 全量轨迹保留：所有任务在Docker沙箱中运行，完整保存执行轨迹、grader产物和环境快照，便于问题回溯。
5. 深度诊断分析：支持按模型规模、模态、任务类型等维度切片，精准定位Harness在技能加载、路径感知等方面的缺陷。

#### 1.4.2 技术原理
采用9个模型×3个Harness×150道任务的三维交叉评测矩阵，通过控制变量法隔离模型与框架的能力贡献。基于五维标签体系实现多维度切片下钻分析，混合评分机制结合确定性规则断言（文件存在性、字段匹配等）与LLM语义评估，平衡客观性与灵活性。依托Docker沙箱实现环境隔离，捕获执行轨迹与工作区产物，通过产物级硬校验避免模型“虚假完工”，确保评测结果可复现。

#### 1.4.3 应用场景
1. Harness开发者优化：开发者通过切片分析，定位框架在技能调用、工具配置等方面的缺陷，针对性优化运行逻辑。
2. 智能体选型参考：用户根据纯文本、多模态等任务类型，选择最佳模型+Harness组合，提升业务场景下的任务完成效率。
3. 基准回归测试：Harness迭代修复后，重新运行对应任务切片，验证优化是否真实解决问题，保障版本迭代质量。
4. 学术研究支撑：为通用智能体研究提供可复现的交叉评测基准，助力模型与框架协同进化的相关研究。

## 2.1 新CopilotKit – 开源 AI Agent 开放框架，全栈 Agent UI 基础设施

CopilotKit是开源的AI原生应用前端开发框架，由AG-UI协议的缔造团队打造，支持三种生成式UI模式，可实现AI Agent与应用前端的双向状态同步、人工介入审批和跨会话持久化，帮助开发者快速构建能生成交互界面的智能助手，一次集成即可在多端运行。

#### 2.1.1 核心功能
1. 三种生成式UI模式：支持受控模式（预定义组件确保品牌一致）、声明式A2UI模式（JSON Schema自动映射组件）、开放式模式（直接生成HTML或驱动可视化工具），覆盖不同开发场景。
2. 双向状态同步：通过WebSocket/SSE长连接实现Agent与前端应用状态的实时双向绑定，Agent可直接修改应用状态，前端自动响应变更。

1. **AG-UI协议层**：定义AI Agent与UI的实时通信标准，通过结构化消息格式实现Agent生成UI指令的标准化传输，完成从对话到界面的范式转换。

3. **双向状态同步架构**：前端通过`useAgent` Hook与Runtime建立长连接，应用状态以共享状态树形式暴露给Agent，Agent修改状态后前端订阅Diff自动重渲染，形成闭环。
4. **人工介入中断机制**：Runtime在执行链中插入中断节点，当Agent调用需确认的工具时，发送中断事件暂停执行流，用户确认后回传结果恢复执行。
5. **持久化会话管理**：所有会话数据按Thread ID存储到内存/Redis/数据库，用户重新接入时Runtime加载完整上下文，Agent从断点恢复执行。

#### 2.1.3 应用场景
1. **SaaS产品内嵌智能助手**：在CRM、ERP、数据分析后台等系统中嵌入Copilot，用户可通过对话直接操作复杂功能，提升产品易用性。

## 2.18 OpenClacky – 李亚飞团队开源的低成本 AI Agent

OpenClacky是李亚飞团队开源的低成本AI Agent，通过精简工具集、智能上下文压缩和模型路由降低使用成本，支持一键安装与跨平台运行。它内置浏览器与IM集成，支持Skill封装与售卖，能让专业人士零代码成为微型SaaS创始人，实现高效低成本的AI辅助工作。

#### 2.18.1 核心功能
1. 智能任务执行：基于16个核心工具，完成编程、数据分析、内容创作等各类工作，提升工作效率。
2. 多模态生成：支持文本、图片、视频等全模态内容创作与编辑，满足多样化内容生产需求。
3. 浏览器自动化：内置浏览器自动抓取网页数据、搜索情报并整理输出，替代人工重复操作。
4. IM工具集成：无缝对接飞书、微信、钉钉等通信工具，任务完成后自动推送结果，实现信息高效流转。
5. Skill市场与封装：支持将行业经验封装为Skill进行分发、更新与售卖变现，打造个人AI技能资产。

#### 2.18.2 技术原理
采用精简工具集架构，仅保留16个核心工具，将长尾能力交给Skill按需调用，减少未使用工具的schema对上下文Token的占用；通过智能上下文压缩技术，在空闲期自动压缩历史对话上下文，降低持续运行成本；基于BYOK模型路由机制，根据任务复杂度动态选择模型，实现精细化成本控制；具备自进化Skill沉淀能力，通过双钩子机制在任务执行中自动沉淀高频操作，减少后续Token消耗；搭配高命中率缓存（官方数据90.6%），降低API调用次数与费用。

#### 2.18.3 应用场景
1. 软件开发：开发者输入自然语言指令，即可自动构建前后端项目，支持代码生成、调试与迭代，降低开发门槛。
2. 内容创作：自媒体运营者通过自然语言生成图文、视频等多模态内容，还能借助Skill实现选题、创作、分发全流程自动化。
3. 数据情报收集：企业员工设置定时任务，让Agent自动搜索行业前沿情报并整理成结构化内容，推送到指定通信工具辅助决策。
4. 专业技能变现：行业专家将专业经验封装为Skill，定价后分发给需求用户，实现知识变现与持续收入。
5. 跨境电商运营：独立站卖家利用AI完成选品分析、建站、投流全链路操作，替代多人团队，降低运营成本。
